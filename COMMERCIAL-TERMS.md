## 3. Project Interpretation of AGPL Obligations

To ensure clarity for commercial integrators, the maintainer adheres to the following interpretations of the AGPL-3.0 regarding **"Combined Works"** and **"Network Interaction"**.

### A. Bundled Distributions & "Stacks"
If you distribute **HiveMind-core** as part of a software bundle (e.g., Docker Compose stack, VM image, hardware appliance) alongside proprietary "skills," "plugins," or "extensions" that are:
1.  **Functionally dependent** on HiveMind-core to operate, AND
2.  **Intimately coupled** via internal protocols or shared runtime logic,

The project considers this a **Combined Work**. Under the AGPL, the source code for the *entire combination* (including proprietary skills) must be made available to users.

**Commercial Note:** If you cannot open-source your proprietary skills/plugins, you must purchase a Commercial License to distribute this stack.

### B. Network Interaction (SaaS / IoT)
Per **AGPL Section 13**, if users interact with HiveMind-core remotely through a computer network (e.g., via a mobile app, web dashboard, or factory control system), you must provide those users access to the Corresponding Source Code.

* This applies even if the software is "deployed" rather than "distributed."
* This applies to any modified versions or "Combined Works" running on your servers/devices.
